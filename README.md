# AI 혁신 발표 투표 시스템

증권사 현업부서들의 AI 활용 혁신 사례 발표에 대한 실시간 투표 시스템입니다.

## 주요 기능

### 참여자 기능
- 🎯 **로그인 없는 영구 세션**: 브라우저 기반 세션 관리로 재방문 시 이전 투표 자동 표시
- ⭐ **별점 투표**: 20개 발표에 대해 1~5점 별점 평가
- 💾 **자동 저장**: 별점 선택 시 즉시 서버에 저장
- ✏️ **투표 수정**: 투표 종료 전까지 언제든지 수정 가능
- 📊 **진행률 표시**: 현재 투표 진행 상황 실시간 확인
- 📱 **모바일 최적화**: 터치하기 쉬운 큰 버튼, 반응형 디자인

### 관리자 기능
- 🔐 **비밀번호 인증**: 간단한 비밀번호로 관리자 페이지 접근
- 📈 **실시간 통계**: 총 참여자 수, 각 발표별 평균 별점, 순위
- 📋 **개별 투표 내역**: 모든 참여자의 투표 내용 조회
- 🎛️ **투표 제어**: 투표 시작/종료 버튼
- 🔄 **자동 새로고침**: 10초마다 통계 자동 업데이트

## 기술 스택

- **프레임워크**: Next.js 14 (App Router)
- **언어**: TypeScript
- **스타일링**: Tailwind CSS

- **데이터베이스**: 메모리 기반 (로컬), Vercel Postgres (프로덕션)

## 발표 데이터 수정 방법

`data/presentations.json` 파일을 수정하여 발표 주제를 변경할 수 있습니다:

```json
{
  "presentations": [
    {
      "id": 1,
      "teamName": "팀명 (10자 이내)",
      "title": "발표 제목 (50자 이내)"
    }
  ]
}
```

**주의사항:**
- `id`는 1부터 시작하는 고유한 숫자
- `teamName`은 10자 이내 권장
- `title`은 50자 이내 권장
- 수정 후 서버 재시작 필요

## 사용 시나리오

### 행사 당일 흐름
1. **사전 준비** (행사 1일 전)
   - `data/presentations.json`을 실제 발표 주제로 수정
   - Vercel에 배포
   - 관리자 페이지에서 정상 작동 확인

2. **행사 시작**
   - 카카오톡 단톡방에 URL 공유
   - 참여자들이 각자의 스마트폰으로 접속하여 투표

3. **행사 진행 중**
   - 관리자는 실시간으로 투표 현황 모니터링
   - 참여자들은 언제든지 투표 수정 가능

4. **행사 종료**
   - 관리자 페이지에서 "투표 종료" 버튼 클릭
   - 최종 순위 확인 및 결과 발표

## 문제 해결

### 투표가 저장되지 않을 때
- 브라우저 localStorage가 활성화되어 있는지 확인
- 시크릿 모드에서는 localStorage가 제한될 수 있음

### 관리자 페이지 접속 불가
- `.env.local` 파일에 `ADMIN_PASSWORD` 설정 확인
- 서버 재시작 후 다시 시도

### 모바일에서 레이아웃 깨짐
- 최신 브라우저 사용 권장 (Chrome, Safari)
- 화면 새로고침 시도
